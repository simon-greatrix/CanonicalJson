# Release Notes — 3.7

## Overview

Version 3.7 is a feature and API-improvement release. The headline changes are a substantial expansion of the `CJArray` and `CJObject` APIs, a new `JsonBuilder` utility class for fluent object construction, and a rename of `CJObject.asJObject` to `asObject`. There are also correctness fixes in the Jackson 2 integration and general code-quality modernisation throughout.

## New Features

### `JsonBuilder` — fluent entry point for building JSON structures

A new `com.pippsford.json.builder.JsonBuilder` utility class provides static factory methods for starting a build operation without having to instantiate `ObjectBuilder` or `ArrayBuilder` directly. All overloads of `add(name, value)` are provided for `JsonValue`, `String`, `BigInteger`, `BigDecimal`, `int`, `long`, `double`, and `boolean`, each returning a new `ObjectBuilder` ready for further chaining.

### `CJArray.EMPTY` and `CJObject.EMPTY` constants

Both classes now expose a public `static final` pre-built, unmodifiable empty instance:
- `CJArray.EMPTY`
- `CJObject.EMPTY`

### Expanded `CJArray` API

Many new typed accessor and mutation methods have been added:

- **Typed `get` accessors** returning canonical subtypes: `getArray`, `getObject`, `getJsonArray`, `getJsonObject`, `getJsonNumber`, `getJsonString`, `getJsonBoolean`.
- **Optional accessors** (`optArray`, `optObject`, `optValue`) returning `null` rather than throwing when an element is absent or the wrong type.
- **Typed `set` methods** accepting `Boolean`, `Number`, `String`, and `JsonValue`; plus `setNull`.
- **Canonical stream/iterator access**: `canonicalStream`, `canonicalParallelStream`, `canonicalIterator`, `canonicalSpliterator` — avoids casting when operating on `Canonical` values directly.
- **`subList`** now returns `CJArray` rather than a plain `List<JsonValue>`.
- **`remove(int)`** now returns `Canonical`.
- Null-checks strengthened on `asArray(Collection)` and `asArrayFromArray(Object)`.

### Expanded `CJObject` API

- **`CJObject.asObject(Map)`** — renamed from `asJObject`; existing code using `asJObject` must be updated.
- **Typed `get` accessors** returning canonical subtypes: `getJsonArray`, `getJsonObject`, `getJsonNumber`, `getJsonString`, `getJsonBoolean`, `getJsonValue` (with default-value overloads).
- **Canonical entry/value views**: `canonicalEntrySet`, `canonicalValues`, and the full set of navigable-map canonical entry methods (`canonicalCeilingEntry`, `canonicalFloorEntry`, `canonicalHigherEntry`, `canonicalLowerEntry`, `canonicalFirstEntry`, `canonicalLastEntry`, `canonicalPollFirstEntry`, `canonicalPollLastEntry`).
- **`compute`, `computeIfAbsent`, `computeIfPresent`** — `Map` compute methods now available.
- **`merge`, `putIfAbsent`, `replace`, `getOrDefault`** — additional `Map` methods now available.
- **Navigable sub-map views** (`headMap`, `descendingMap`) now return `CJObject`.
- Use of Java pattern-matching `instanceof` throughout.

## Breaking Changes

- **`CJObject.asJObject(Map)`** has been renamed to **`CJObject.asObject(Map)`**. Update all call sites.

## Bug Fixes and Improvements

### Jackson 2 integration

- **`CanonicalSerializer`** — fixed handling of polymorphic types: when a type discriminator field has already been written into an object context, the serialiser now correctly uses field-by-field output rather than `writeRawValue`, avoiding a `JsonGenerationException`.
- **`JacksonReader`** — updated to use the non-deprecated `currentTokenLocation()` in place of `getTokenLocation()`; correctly handles the case where the parser is already positioned inside an object (e.g. after a Jackson type discriminator has been consumed).
- **`CanonicalFactory`** — required and forbidden feature sets are now expressed as `EnumSet` constants for clarity and maintainability.
- **`Convert`** — expanded with additional `toJackson` convenience overloads returning `ArrayNode`, `ObjectNode`, and `ContainerNode`; uses modern `node.properties()` iteration.

### Jackson 3 integration

- `JacksonReader` minor alignment fixes.
- `CanonicalGenerator` minor fix.

### IJSON

- Removed the precomputed `POWERS_OF_TEN` table (was unused after earlier refactoring).
- Minor whitespace and style fixes.

### General

- `switch` statements modernised to switch expressions throughout (`CanonicalCreator`, `JacksonReader`, `Test` patch operation, `IJsonNumberSerializer`).
- `PatchOperation` minor fix.
- `CJString` Javadoc corrected.
- `package-info.java` files updated.
