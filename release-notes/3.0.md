# Release Notes — 3.0

**Released:** 2024-09-25

## Overview

Version 3.0 is a major release that migrates the library from the `javax.json` API to the `jakarta.json` API, reflecting the broader Java ecosystem's transition to the Jakarta EE namespace. This is a **breaking change** for any code that depends on the `javax.json` imports.

## Breaking Changes

- **Jakarta EE migration:** All `javax.json.*` imports are replaced with `jakarta.json.*`. Consumers must update their import statements and ensure they are using a Jakarta-compatible JSON API dependency.
- The service provider registration file has been renamed from `javax.json.spi.JsonProvider` to `jakarta.json.spi.JsonProvider`.

## Changes

- Upgraded all source files and build configuration to target the `jakarta.json` API.
- Javadoc improvements and checkstyle rule updates across the codebase.
- Improved build configuration and publishing setup.
- Expanded `Generator` with additional capabilities.
- Added `MutableLocation` for tracking parse position.
- Added `NonFiniteNumberException` and `NotJsonException` exception types.
- Added `CanonicalCreator` as a dedicated factory/creator class.
- Added `FormattedJson` interface.
- Added `ByteArrayGenerator`, `NoOpFormatter`, `PrettyFormatter`, and `Utf8Appendable` I/O helpers.
- Upgraded to JUnit 5 for the test suite.
- Expanded parser to support reading multiple root structures from a single stream (configurable via `ParserFactory.REQUIRE_SINGLE_ROOT`).
- Numerous code quality improvements and Javadoc additions throughout.
