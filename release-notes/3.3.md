# Release Notes — 3.3

**Released:** 2025-08-20

## Overview

Version 3.3 introduces IJSON (Internet JSON, [RFC-7493](https://www.rfc-editor.org/rfc/rfc7493)) support — a profile of JSON that restricts numbers to those representable without loss of precision in JavaScript environments. It also removes large sample data files from the repository.

## New Features

- **IJSON support:** Added `IJson` interface and `IJsonNumberSerializer` in the new `com.pippsford.json.ijson` package. IJSON (RFC-7493) restricts JSON numbers to IEEE 754 double-precision safe integers and finite values, ensuring interoperability with JavaScript consumers that cannot represent arbitrary-precision numbers.
  - `IJsonNumberSerializer` provides Jackson-compatible serialisation.
  - `ForbiddenIJsonException` added for constraint violations specific to the IJSON format.
  - Comprehensive test coverage via `IJsonTest` and `IJsonNumberSerializerTest`.
- **`ForbiddenJsonException`** added to the exception hierarchy for cases where a JSON value violates format constraints.
- Extended `NumberParser` tests for additional edge cases.

## Other Changes

- Large binary/sample test data removed from the git repository (now excluded via `.gitattributes` and `.gitignore`).
- `JsonModule` (Jackson integration) updated.
- `Test` patch operation minor update.
- `PrettyFormatterTest` and `NumberParserTest` expanded.
- Build dependency updates.
