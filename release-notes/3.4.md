# Release Notes — 3.4

**Released:** 2025-11-23

## Overview

Version 3.4 adds Java Serialization support to the canonical JSON primitive types and refines IJSON number handling. The `BadNumber` placeholder class has been removed in favour of proper exception handling.

## New Features

- **Java Serialization:** `CJTrue`, `CJFalse`, `CJNull`, `CJString`, `CJBigDecimal`, `CJBigInteger`, `CJInt`, `CJLong`, and `CJJson` now implement `java.io.Serializable`, enabling canonical JSON values to participate in standard Java object serialisation. A new `SerializationTest` validates round-trip correctness.
- **`CodePointOrder`** utility class added to provide Unicode code-point-order comparisons, used internally for canonical key ordering.
- **IJSON improvements:** `IJsonNumberSerializerTest` significantly expanded with randomised number tests using a new `Xoshiro256pp` pseudorandom number generator. JavaScript reference test scripts added under `src/test/js/ijson/` for cross-language validation.

## Removed

- **`BadNumber`** class removed. Invalid number states are now handled via proper exception paths rather than a sentinel object.

## Other Changes

- `CJArray` extended with additional list-manipulation methods.
- `CJObject` refactored for correctness and clarity.
- `ArrayBuilder` and `ObjectBuilder` updated.
- `TrustedGenerator` minor fix.
- `NumberParser` updated to remove dependency on `BadNumber`.
- `CJObjectTest` updated to reflect the refactored `CJObject`.
